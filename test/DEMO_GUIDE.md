# 系统功能演示指南

## 系统功能概览

### 管理员（11项功能）
| # | 功能 | 说明 |
|---|------|------|
| 1 | 科室管理 | 增删改科室，分配病房和药品 |
| 2 | 病房管理 | 增删改病房，变更归属科室 |
| 3 | 医生管理 | 增删改医生，设置级别/科室/出诊日 |
| 4 | 药品管理 | 增删改药品，管理库存 |
| 5 | 患者管理 | 增删改患者（门诊/住院） |
| 6 | 挂号管理 | 为患者挂号（自动校验医生出诊/重复挂号） |
| 7 | 住院管理 | 入院登记（分配床位）→ 出院结算 |
| 8 | 药房管理 | 入库/出库/查看库存 |
| 9 | 报表查询 | 药品/医生/床位/资金/工作量/营业额统计 |
| 10 | 时间推进 | 模拟天数流逝，自动扣住院费、过期挂号 |
| 11 | 保存数据 | 持久化到文件 |

### 医生（7项功能）
| # | 功能 | 说明 |
|---|------|------|
| 1 | 查看待诊患者 | 列出挂号为本人且待就诊的患者 |
| 2 | 接诊 | 填写主诉/诊断，创建看诊记录 |
| 3 | 开具处方 | 选择药品+数量，生成处方 |
| 4 | 开具检查 | 开具检查项目 |
| 5 | 查看住院患者 | 查看本人负责的住院患者详情 |
| 6 | 为住院患者诊治 | 为住院患者开处方/检查 |
| 7 | 查询报表 | 个人门诊/住院/处方统计 |

### 患者（3项功能）
| # | 功能 | 说明 |
|---|------|------|
| 1 | 查看就诊记录 | 挂号→看诊→检查→处方→住院全部记录 |
| 2 | 支付费用 | 支付挂号费/处方药费 |
| 3 | 查看住院信息 | 仅住院患者：病房/床位/押金/每日明细 |

## 演示场景

### 场景一：管理员基础操作（5分钟）

**目的**：展示管理员各项增删改查功能

```
1. 管理员登录
   ID: admin  密码: 123456

2. 科室管理(1) → 添加科室(1)
   - 新增一个科室: 名称=康复科, 描述=提供康复治疗服务
   → 观察自动生成的科室ID

3. 医生管理(3) → 添加医生(1)
   - 姓名=赵康复, 级别=2(主治医师), 科室ID=刚创建的康复科ID
   - 出诊日: 1,3,5 (周一三五)
   → 观察医生ID和出诊日显示

4. 药品管理(4) → 查看全部(4)
   → 观察25种默认药品的库存

5. 药品管理(4) → 入库(5)
   - 选择阿莫西林(MED001), 入库100盒
   → 观察库存从500变为600

6. 保存数据(11)
   → 提示"数据保存成功"
```

**关键观察**：ID自动生成、输入校验（空值/重复/不存在ID的报错提示）

---

### 场景二：挂号→接诊→开药→缴费 完整流程（10分钟）

**目的**：演示患者从挂号到缴费的完整门诊流程

```
第一步：管理员挂号
  管理员登录 → 挂号管理(6) → 创建挂号(1)
  - 先选患者(选列表中的一个门诊患者，如PAT005)
  - 再选科室(选内科 DEPT001)
  - 系统自动筛选当日出诊的内科医生
  - 选择一个医生(如DOC001 张伟明)
  → 挂号成功，记下挂号ID(如 R-0001-016)

第二步：医生接诊
  退出到主菜单 → 医生登录(2) → 输入DOC001
  - 查看待诊患者(1) → 看到刚挂号的PAT005
  - 接诊(2) → 输入挂号ID
  - 主诉: 头痛发热三天，伴有咳嗽
  - 诊断: 上呼吸道感染
  → 接诊成功，记下就诊ID

第三步：医生开处方
  开具处方(3) → 输入就诊ID
  - 系统显示该科室可用药品列表
  - 输入药品ID: MED001(阿莫西林), 数量2, 单价15
  - 继续添加: MED004(对乙酰氨基酚), 数量1, 单价10
  - 输入0结束添加
  → 处方总金额: 40元, 记下处方ID

第四步：患者缴费
  退出到主菜单 → 患者登录(3) → 输入PAT005
  - 查看就诊记录(1) → 可看到挂号、看诊、处方记录
  - 支付费用(2) → 看到待缴挂号费(20元)和处方费
  - 选1=全部支付
  → 支付成功，医院资金增加，药品库存扣减
```

**关键观察**：不同角色视角的数据一致性、支付后库存变化

### 场景三：住院管理流程（10分钟）

**目的**：演示入院→每日扣费→出院的住院全流程

```
第一步：办理入院
  管理员登录 → 住院管理(7) → 办理入院(1)
  - 输入一个门诊患者ID(如PAT010)
  - 系统显示该患者最近的挂号科室及对应病房
  - 查看可用床位 → 选择床位号
  - 选择主管医生(可多选)
  - 输入押金: 5000元
  → 入院成功！患者类型变为"住院"

第二步：时间推进（模拟住院3天）
  时间推进(10)
  - 输入3天
  → 系统自动扣除3天×200元=600元押金
  → 显示剩余押金4400元
  → 过期挂号自动标记为"已过期"

第三步：患者查看住院信息
  退出 → 患者登录(3) → 输入PAT010
  - 查看住院信息(3)（此选项仅住院患者可见）
  → 查看病房、床位、主管医生、每日费用明细、押金余额

第四步：出院结算
  退出 → 管理员登录 → 住院管理(7) → 办理出院(2)
  - 输入住院ID或患者ID
  - 系统显示：住院天数、总费用、押金、应退/应补金额
  - 确认出院
  → 床位释放，患者类型恢复为门诊
```

**关键观察**：押金实时变化、每日扣费明细、出院费用核算

### 场景四：时间推进与过期处理（5分钟）

**目的**：展示时间流逝对系统数据的影响

```
准备：先为3个不同患者挂号（参考场景二第一步）
  管理员 → 挂号管理 → 分别为PAT020, PAT030, PAT040挂号
  记下这些挂号记录

步骤1：推进时间5天
  时间推进(10) → 输入5天
  → 观察输出：
    - 5天×200元×35名住院患者=35000元从押金扣除
    - 刚创建的3条挂号全部标记为"已过期"
    - 当前日期从 Day1 变为 Day6

步骤2：验证过期效果
  退出 → 医生登录 → 输入之前挂号的医生ID
  - 查看待诊患者(1) → 之前挂号的3个患者不再出现（已过期）

步骤3：验证住院扣费
  退出 → 患者登录 → 输入任意住院患者ID(如PAT080)
  - 查看住院信息(3) → 押金已扣除5天费用，余额减少
```

**关键观察**：过期机制、每日自动扣费、时间显示变化

### 场景五：报表查询（5分钟）

**目的**：展示各维度数据统计

```
管理员登录 → 报表查询(9)

1. 药品查询(1)
   - 输入 all → 显示全部25种药品
   - 选择排序: 5(按名称字母序)
   → 观察药品按商品名排序

2. 医生信息查询(2)
   - 输入 DEPT001(内科)
   → 查看内科所有医生及出诊安排

3. 病房床位占用情况(3)
   - 输入 DEPT001(内科)
   → 查看内科病房每张床位的占用/空闲状态

4. 医院资金统计(5)
   → 查看挂号费、药费、住院费三项收入构成

5. 医生工作量统计(6)
   - 起始天数: 0, 结束天数: 9999
   → 查看每位医生的接诊量和处方数排名

6. 营业总额统计(7)
   - 起始天数: 0, 结束天数: 9999
   → 查看总营收及各项占比
```

**关键观察**：多维度统计、排序筛选、资金构成

### 场景七：数据持久化验证（3分钟）

**目的**：证明数据可以保存并在重启后恢复

```
第一步：
  管理员登录 → 做个明显修改（如新增一个科室"验证用科室"）
  → 保存数据(11)
  → 退出系统(0)

第二步：
  重新启动 hospital.exe
  → 观察系统启动信息："数据加载成功"（而非"使用默认初始数据"）
  → 管理员登录 → 科室管理 → 确认"验证用科室"存在
```

**关键观察**：关闭后数据不丢失、文件序号连续性

### 场景八：健壮性测试（3分钟）

**目的**：展示系统的错误处理能力

```
尝试以下操作，观察系统如何提示错误而非崩溃：

1. 输入类型错误
   管理员登录 → 药品管理 → 入库
   - 选择药品后，数量输入"abc"（非数字）
   → [错误] 输入不是有效整数，请重新输入。

2. ID不存在
   医生登录 → 输入"DOC999"
   → [错误] 医生ID不存在！

3. 库存不足
   管理员 → 药品管理 → 出库
   - 选择药品，数量输入99999
   → 库存不足! 当前库存仅...

4. 重复挂号
   管理员 → 挂号管理 → 对已挂号的患者在同一科室重复挂号
   → 该患者今天已在该科室挂号，不可重复挂号!

5. 床位已满
   当病房床位全部占用后尝试办理入院
   → 床位分配失败/无可用床位

6. 删除有依赖的科室
   管理员 → 科室管理 → 删除内科(有医生和病房)
   → 科室内科下还有X名医生，无法删除!

7. 登录密码错误
   管理员登录 → 输入错误密码
   → 密码错误! 剩余X次尝试机会
```

## 演示顺序建议（总时长约40分钟）

| 顺序 | 场景 | 时长 | 累计 |
|------|------|------|------|
| 1 | 场景一：管理员基础操作 | 5min | 5min |
| 2 | 场景二：挂号→接诊→开药→缴费 | 10min | 15min |
| 3 | 场景三：住院管理流程 | 10min | 25min |
| 4 | 场景四：时间推进与过期处理 | 5min | 30min |
| 5 | 场景五：报表查询 | 5min | 35min |
| 6 | 场景六：批量导入数据 | 3min | 38min |
| 7 | 场景七：数据持久化验证 | 3min | 41min |
| 8 | 场景八：健壮性测试 | 选做 | - |

建议开场用**场景二**（患者完整流程）快速展示系统的核心价值，然后用**场景三**展示住院管理，最后用**场景四+五**展示自动化处理和数据统计能力。
